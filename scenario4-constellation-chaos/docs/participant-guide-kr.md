# 참가자 가이드 (한국어), 위성을 군집위성에 충돌시켜라

> 확인용 한국어판입니다. 공식 배포본은 영어 `participant-guide.md`이며, 영어판이
> 기준(source of truth)입니다.

환영합니다, 오퍼레이터. 지금 당신은 위성의 명령 업링크에 접근할 수 있습니다.
당신이 보내는 모든 것은 **실제로 통용되는 정상 명령 포맷**입니다. 단, 값은 당신이 정합니다.

목표 위성은 **DEMOSAT**. **AURORA 군집위성**(조금 더 높은 궤도를 함께 도는 45기의
위성 링) 바로 아래를 도는 위성 하나입니다. DEMOSAT는 아주 작은 **station-keeping**
추진 분사로 제 자리를 지킵니다. 당신의 임무는 그 분사 하나를 악용해 DEMOSAT를 링까지
끌어올려 이웃 위성과 충돌시키는 것입니다. 콘솔 왼쪽의 **TARGET INTEL** 도시어를
읽으세요. 필요한 정답이 거기 있습니다.

당신은 화면 두 개로 작업합니다. **모니터 1**(앞에 있는 화면)에서 계획하고 제작하며,
**모니터 2**(큰 화면)에서 위성과 지상국을 보며 결과를 관찰합니다.

---

## 1단계, 추진값 계산 (궤도 플래너)

```
  ORBIT PLANNER를 보세요. DEMOSAT(안쪽 궤도)가 AURORA 링(바깥쪽) 아래에 있습니다.
  PROGRADE 추진값(delta-v, m/s)을 입력창이나 +/- 버튼으로 올리세요.
  값을 올릴수록 예상 궤도가 위로 상승합니다.

  너무 작으면 -> 궤도가 링에 못 미침 (SHORT).
  너무 크면   -> 궤도가 링 위로 넘어감 (OVERSHOOT).
  딱 맞으면   -> 상태가  COLLISION COURSE.   (약 15에서 30 m/s)
```

숫자를 바꾸면 예상 궤도가 실시간으로 갱신됩니다. Prograde는 DEMOSAT를 진행 방향으로
밀어 궤도의 반대편(원지점)을 끌어올립니다. Radial(두 번째 값)은 0으로 두면 됩니다.
충돌 코스가 되는 prograde 값을 기억해 두세요. 4스텝의 STEP 3에서 입력합니다.

---

## 2단계, 업링크 조립 (4개 모두 잠겨야 함)

```
  STEP 1 - TARGET ADDRESSING (타깃 주소 지정)
    도시어의 DEMOSAT와 일치하는 Spacecraft ID(SCID)를 고르세요.
    잘못된 ID -> 엉뚱한 위성 -> 명령이 무시됩니다.

  STEP 2 - COMMAND SELECT (명령 선택)
    서브시스템과 명령 선택:  [ PROP ★ ] -> orbit_maneuver
    (위성의 station-keeping 추진 분사)

  STEP 3 - COMMAND VALUE (명령 값)
    플래너에서 찾은 PROGRADE delta-v(m/s)를 입력하고 RADIAL은 0으로 둔 뒤,
    CONFIRM VALUE를 누르세요. 안전한 분사는 겨우 몇 m/s이므로, 당신의 값은
    정상 station-keeping 분사를 명백히 벗어난 값입니다.

  STEP 4 - RF CONFIG (RF 설정)
    변조, baud, sample rate를 위성 수신기에 맞추세요(도시어 참고):
    OOK, 100 bps, 24 kSa/s.
    잘못된 RF -> 위성이 신호를 복조하지 못합니다.

  -> 4개가 모두 잠기면 UPLINK ARMED. [ GENERATE UPLINK IQ ]를 누르세요.
    당신의 명령이 무선 신호 파일(attack.cf32)로 변환됩니다.
```

오른쪽 **CCSDS PACKET**이 진행에 따라 바이트 단위로 조립되는 걸 관찰하세요. 실제 위성
텔레커맨드가 만들어지는 방식 그대로입니다. SCID를 바꾸면 바이트가 바뀝니다. 엉뚱한
위성을 지정하면 완전히 다른 메시지가 되죠.

---

## 3단계, 전송하고 관찰하기

```
  [ TRANSMIT UPLINK -> DEMOSAT ]를 누르세요.
  이제 모니터 2를 보세요.
  DEMOSAT가 추진기를 점화하고 궤도가 AURORA 링을 향해 상승합니다.
  링에 도달해 한 위성과 충돌합니다.
  잔해가 흩어지고, 지상국이 CONSTELLATION COLLISION을 외칩니다.
```

빗나가면 모니터 2에 "no collision, reset to retry"가 뜹니다. 콘솔의 **RESET
SIMULATION**을 누르고 플래너로 돌아가 prograde 값을 다시 조정하세요.

---

## 방금 무슨 일이 일어났나?

이 궤도 수정 명령은 **완전히 정상적인 명령**입니다. 운영자들이 위성을 제 자리에
유지하려고 매일 사용하죠. 당신은 어떤 프로토콜도 깨지 않았고, 어떤 보안도 우회하지
않았습니다. 그저 **정상 명령에 악용된 값**을 실었을 뿐입니다.

작은 분사는 DEMOSAT를 제 자리에 둡니다. 당신의 과도한 분사는 궤도를 AURORA 링까지
끌어올렸고, 거기서 이웃 위성과 충돌했습니다. 그리고 피해는 두 위성에서 끝나지 않습니다.
충돌은 계속 궤도를 도는 **잔해** 구름을 만들고, 이 잔해가 군집의 **다른 위성들**까지
때릴 수 있습니다.

**이것이 핵심입니다: 위성 하나의 문제가 여러 위성으로 번진다.** 위성으로 가득 찬
하늘에서 악용된 명령 하나가 모두의 문제가 됩니다. 그 명령이 어떻게 계획되고 전달되는지
이해하는 것이 방어의 첫걸음입니다.

---

## 더 해볼 것
- prograde delta-v를 **안전한 몇 m/s**로 되돌려 다시 생성해 보라. 정상 station-keeping
  분사다. 플래너 상태와 (충돌이 없는) 결과를 비교해 보라.
- 일부러 **잘못된 SCID나 RF 설정**을 골라 업링크가 다시 잠기는 걸 보라. 명령이
  전달될 수 없다.
- 링에 맞추는 대신 일부러 **링 위로 넘어가는** 분사를 시도해 보라. 그다음 리셋하고
  다시 충돌 코스로 값을 낮춰 보라.

*명령 구조에 대한 설명이 필요하면 부스 운영자에게 문의하세요.*
