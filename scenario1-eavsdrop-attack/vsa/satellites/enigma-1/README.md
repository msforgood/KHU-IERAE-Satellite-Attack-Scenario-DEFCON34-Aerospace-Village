# ENIGMA-1 — Scenario 1 (Eavesdropping)

Fictional LEO Earth-observation cubesat used by the booth's **Eavesdropping**
demo. Visitor tunes VSA4LV to the satellite's downlink, captures the signal,
and decodes the broadcast image with a gr-satellites GNU Radio flowgraph.

This satellite is **downlink-only** — no uplink, no command interface, no
attack vector. It exists in VSA4LV purely as a target for passive intercept.

## RF spec

| Parameter        | Value                                       |
|------------------|---------------------------------------------|
| Center frequency | 433.500 MHz (UHF)                           |
| Modulation       | GFSK, G3RUH 9600, ±2.4 kHz, Gaussian BT = 0.5 |
| Symbol rate      | 9600 baud                                   |
| IQ sample rate   | 96 000 Hz (10 samples/symbol, native)       |
| Polarization     | RHCP                                        |
| EIRP             | 120 (VSA-normalised units)                  |

## On-the-wire stack (AX.25 / G3RUH 9600)

Top of the stack is the image; bottom is the radio symbols.

```
image bytes (192×128 grayscale)
  → DEFLATE (zlib) compression   (4-byte length prefix + stream)
  → split into 200-byte chunks, each prefixed with a 2-byte sequence number
  → AX.25 UI frames              (dest CQ / src ENGMA1, ctrl 0x03, PID 0xF0)
  → HDLC framing                 (0x7E flags, bit-stuffing, CRC-16 FCS)
  → G3RUH scrambler              (1 + x^12 + x^17)
  → NRZI encode
  → GFSK on 433.5 MHz
```

The compressed image spans ~29 AX.25 frames; the burst is ~6.2 s, then the IQ
file pads 1 s of silence before the cycle loops (7.22 s loop).

## Orbit

LEO sun-synchronous, ~700 km, inclination 98.5°, period ~98 min.
NORAD ID 90001 (fresh fictional range — does not collide with mrradio).

## GNU Radio decode chain (gr-satellites)

The booth ships a ready flowgraph
([`vsa4lv-challenges/scenario-1/solution/enigma1_decoder.grc`](../../../vsa4lv-challenges/scenario-1/solution/enigma1_decoder.grc)).
The chain is the amateur-satellite standard 9600 G3RUH AX.25 receiver:

1. **File Source** (or VSA4LV stream) — cf32
2. **Throttle** + **Freq Xlating FIR** (tune here for Doppler)
3. **gr-satellites: FSK Demodulator** — baudrate `9600`, deviation `2400`, IQ input `True`
4. **gr-satellites: AX.25 Deframer** — **`g3ruh_scrambler = True`**
5. Per-frame PDU → **image reassembler** (embedded block): order Info fields by
   sequence, strip the 4-byte length prefix, DEFLATE-inflate → 192×128 PNG

Wrong baudrate, or G3RUH scrambler left off ⇒ no frames decode. All correct ⇒
picture appears. gr-satellites' clock recovery handles any capture sample rate,
so a recording at 96 k / 100 k / etc. all decode (set the flowgraph `samp_rate`
to the recording's `core:sample_rate`).

## Signal file

The `.cf32` (hash-verified by VSA4LV's loader) is generated by
[`vsa4lv-challenges/scenario-1/signal_gen/generate_enigma1.py`](../../../vsa4lv-challenges/scenario-1/signal_gen/generate_enigma1.py)
and its SHA-256 is registered in [`vsa4lv/electron/config.js`](../../electron/config.js)
as `iqFileHash`. Re-run the generator any time the image content changes; copy
the new SHA-256 into `config.js`.

The generator builds the AX.25 frames with the exact gr-satellites blocks
(`hdlc_framer`, `scrambler_bb(0x21,0,16)`, `nrzi_encode`) and **self-tests by
decoding its own output through gr-satellites' `ax25_deframer` before writing** —
so a broken signal never ships.
