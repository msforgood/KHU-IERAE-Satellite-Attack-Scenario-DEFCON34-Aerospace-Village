### 시연 단계

```
① 명령 조립          ② IQ 생성        ③ 위성 조준·송신          ④ 드론 스푸핑            ⑤ 피해(지속)
Command Builder ─▶ attack.cf32 ─▶ gpredict+VSA 조준 후    ─▶ 드론이 위성인 척       ─▶ 위성은 계속
웹 화면             (IQ 신호파일)     VSA가 IQ 송신(uplink)      "정상" 비콘을 GS에         tumbling · 모터 회전
(4스텝 퍼즐)                         └▶ 🛰️ 안테나 · ☀️ 솔라패널   흘려 경보만 끔             (물리 피해 유지)
                                        모터 동작 + GS 경보       └▶ GS 대시보드 → NOMINAL
```

| 단계               | 어디서                                       | 물리 모터 / 효과                          |
| ------------------ | -------------------------------------------- | ----------------------------------------- |
| **① 명령 조립**    | Command Builder 웹 (`:8000`)                 | —                                         |
| **② IQ 생성**      | Command Builder `GENERATE` → `attack.cf32`   | —                                         |
| **③ 위성 조준·송신** | gpredict + OpenVSA(VSA) `TRANSMIT`           | 🛰️ 안테나 스윕 · ☀️ 솔라패널 무한 회전 + GS 경보 |
| **④ 드론 스푸핑**  | Drone 콘솔 (`:8000` `/drone`) → `/api/spoof` | GS 대시보드만 NOMINAL로 위장 (경보 꺼짐)   |
| **⑤ 피해(지속)**   | victim 지상국 대시보드 (`:4540`)             | 위성 실제 상태·모터는 계속 공격 상태 유지 |

### 핵심 포인트 — ④ 드론 스푸핑

- 드론은 **하드웨어로 연결하지 않는다.** 위성인 척(`SAT-200`) 가짜 "정상" 비콘을 GS에 흘리는 것을 **소프트웨어로 시뮬레이션**한다.
- GS 백엔드는 텔레메트리를 **두 갈래**로 내보낸다:
  - `/api/state` (아두이노 브리지·물리 모터) → **항상 실제 상태**(tumbling 유지) = 위성은 계속 돈다.
  - 대시보드 WebSocket → 스푸핑 ON이면 **드론이 위조한 NOMINAL**(배터리 ~98%, SUN-TRACKING, COMM CONNECTED)을 대신 송출 → **경보만 사라짐.**
- `POST /api/spoof {on:true}` 로 켜고, `{on:false}` 또는 `/api/reset` 로 진실 복원.
- 드론 콘솔은 victim 대시보드를 iframe으로 임베드 → 경보가 **RED → GREEN** 으로 뒤집히는 걸 그 자리에서 확인.
